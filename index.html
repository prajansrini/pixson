About Pixson & File Encodings

What is Pixson?

Pixson is a universal file encoder that acts as a "disguise" for your files. Sometimes, you need to share a file (like an image, PDF, or video) on a platform or system that only accepts plain text or doesn't support file attachments. Pixson solves this by converting any file into a compact string of text.

You can encode a file into text, copy it, paste it anywhere (like an email, forum, or code snippet), and the recipient can use Pixson's Decode tab to instantly rebuild the original file. It is 100% client-side, meaning your files never leave your browser.

Containers (The "Box")

PXSN Binary (.pxsn): The newest, fastest format. It creates the smallest possible files and is highly recommended for all your saves.

JSON Text (.json): A standard text format. Great if you need to copy and paste your file as pure text, but it makes the file size a bit larger.

GZipped JSON (.json.gz): Identical to JSON Text but the whole file is compressed, so it's smaller but can't be copy-pasted as plain text.

Encodings for Images (The "Data")

Pixen / Pixen Ultra: Custom compression algorithms built specifically for Pixson! They are dramatically smaller than standard formats for pixel art and flat-color images.

WebP Lossless: The standard choice for photographs and highly detailed, colorful images.

Base64 / Hex / Packed / Int: Alternative text-based formats. Usually, you can ignore these unless a specific app or project requires them.

Encodings for Generic Files (The "Data")

GZip (Raw Bytes): A highly efficient compression method that shrinks your files as small as possible. (Only available when using the PXSN v2 container).

Base64 / Base85 Text: Converts your file into a long string of standard text characters. Useful if you need to send a file through a system that only accepts text.
